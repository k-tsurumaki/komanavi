# KOMANAVI 実力測定用 推奨行政ドキュメント

## 選定基準

KOMANAVIの能力を総合的に測定するため、以下の観点で難易度を設定：

1. **構造の複雑さ** - 条件分岐、例外規定の多さ
2. **専門用語の量** - 平易化が必要な用語の数
3. **対象者の特定困難度** - 「自分が該当するか」の判断難易度
4. **手続きの複雑さ** - 必要書類、ステップ数
5. **情報量** - ページの文字数、セクション数

## 🟢 レベル1：入門編（基本機能確認）

### 1-1. 児童手当
- **URL例**: `https://www.cfa.go.jp/policies/kokoseido/jidouteate/`
- **難易度**: ★☆☆☆☆
- **特徴**:
  - 構造がシンプルで対象者が明確
  - 所得制限という条件分岐が1つ
  - 手続きフローが直線的
- **測定ポイント**:
  - 基本的な制度概要の抽出
  - 対象者条件の正確な把握
  - 申請手続きの整理

### 1-2. マイナンバーカード申請
- **URL例**: `https://www.kojinbango-card.go.jp/`
- **難易度**: ★☆☆☆☆
- **特徴**:
  - 全国民が対象で条件分岐が少ない
  - 複数の申請方法がある（選択肢の整理）
  - 必要書類が明確
- **測定ポイント**:
  - 複数の手続き方法の整理
  - ステップバイステップの抽出

## 🟡 レベル2：中級編（実用性検証）

### 2-1. 高額療養費制度
- **URL例**: `https://www.mhlw.go.jp/stf/seisakunitsuite/bunya/kenkou_iryou/iryouhoken/juuyou/kougakuiryou/`
- **難易度**: ★★★☆☆
- **特徴**:
  - 所得区分による自己負担限度額の変動
  - 世帯合算、多数回該当などの複雑なルール
  - 「結局いくら戻ってくる?」の計算が困難
- **測定ポイント**:
  - 複雑な条件分岐の整理
  - 計算ロジックの可視化
  - 例外条件（70歳以上など）の抽出

### 2-2. 育児休業給付金
- **URL例**: `https://www.mhlw.go.jp/stf/seisakunitsuite/bunya/0000158500.html`
- **難易度**: ★★★☆☆
- **特徴**:
  - 雇用保険加入条件、勤続期間条件
  - 給付率の変動（最初6ヶ月67%、以降50%）
  - パパ・ママ育休プラスなどの特例
- **測定ポイント**:
  - 時系列の条件変化の表現
  - 複数の給付パターンの整理
  - 専門用語（賃金日額等）の平易化

### 2-3. 住宅ローン減税
- **URL例**: `https://www.mlit.go.jp/jutakukentiku/house/jutakukentiku_house_tk2_000017.html`
- **難易度**: ★★★☆☆
- **特徴**:
  - 物件種別、入居時期による控除額の違い
  - 所得制限、床面積要件など複数条件
  - 13年間という長期の制度設計
- **測定ポイント**:
  - 年度による制度変更の整理
  - 複雑な計算式の可視化
  - 必要書類の多さへの対応

## 🔴 レベル3：上級編（限界性能測定）

### 3-1. 生活保護制度
- **URL例**: `https://www.mhlw.go.jp/stf/seisakunitsuite/bunya/hukushi_kaigo/seikatsuhogo/seikatuhogo/`
- **難易度**: ★★★★☆
- **特徴**:
  - 8種類の扶助（生活・住宅・教育等）
  - 世帯状況、地域、年齢による基準額の違い
  - 資産調査、扶養義務者調査など複雑な審査
  - スティグマがあり「自分が対象か」の判断が心理的に困難
- **測定ポイント**:
  - 多層的な条件分岐の整理
  - センシティブな情報の扱い
  - 漫画での心理的ハードル低減効果

### 3-2. 障害年金
- **URL例**: `https://www.nenkin.go.jp/service/jukyu/shougainenkin/jukyu-yoken/index.html`
- **難易度**: ★★★★★
- **特徴**:
  - 初診日、保険料納付要件など複雑な受給要件
  - 障害等級による給付額の違い
  - 障害認定基準が抽象的
  - 「自分の症状が該当するか」の判断が極めて困難
- **測定ポイント**:
  - 抽象的な認定基準の具体化
  - 医療用語の平易化
  - 複数の前提条件の組み合わせ処理

### 3-3. 相続税申告
- **URL例**: `https://www.nta.go.jp/taxes/shiraberu/taxanswer/sozoku/4152.htm`
- **難易度**: ★★★★★
- **特徴**:
  - 基礎控除、配偶者控除、小規模宅地特例など複雑な控除
  - 財産評価、債務控除など専門的計算
  - 10ヶ月という期限と罰則
  - 専門家への依頼が前提の複雑さ
- **測定ポイント**:
  - 極めて専門的な内容の要約限界
  - 「専門家に相談すべき」という判断の提示
  - 期限と罰則の明確化

## 🎯 推奨テストシナリオ

### フェーズ1：基本機能検証
1. **児童手当**で基本的な抽出・要約・漫画生成をテスト
2. エラーハンドリング、UI/UXの確認

### フェーズ2：実用性検証
1. **高額療養費制度**で複雑な条件分岐への対応を確認
2. **育児休業給付金**で時系列変化の表現を確認
3. ユーザビリティ（理解度向上効果）の測定

### フェーズ3：限界性能測定
1. **生活保護制度**でセンシティブな内容への対応を確認
2. **障害年金**で抽象的基準の具体化能力を測定
3. システムの限界を把握し、改善点を特定

## 📊 評価指標

各ドキュメントで以下を測定：

### 定量指標
- [ ] URL解析時間（目標: 10秒以内）
- [ ] 要約生成時間（目標: 15秒以内）
- [ ] 漫画生成時間（目標: 60秒以内）
- [ ] 中間表現の完全性（必須項目の抽出率）

### 定性指標
- [ ] 要約の正確性（原文との照合）
- [ ] 要約の平易性（中学生レベルで理解可能か）
- [ ] 漫画のストーリー性（制度理解につながるか）
- [ ] チェックリストの実用性（次アクションが明確か）

### ユーザー評価（可能であれば）
- [ ] 「自分が対象か分かった」
- [ ] 「次に何をすべきか分かった」
- [ ] 「元のページより理解しやすい」

## 💡 テスト時の注意点

1. **免責事項の確認**: 生成内容が正確でない可能性を明示
2. **元サイトへの誘導**: 必ず公式情報を確認するよう促す
3. **著作権への配慮**: スクレイピング対象サイトの利用規約確認
4. **ロボット対策**: robots.txt の尊重、適切なUser-Agent設定
