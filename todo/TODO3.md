# KOMANAVI Phase 3

## 目的
リポジトリの現状実装に対して、本当に実施すべきタスクを優先度順に整理する。

## P0（必須）

### システムアーキテクチャ
- [ ] システムアーキテクチャの第一稿を作成

### 認証・個人データ・パーソナライズ漫画
- [ ] ログイン機能の実装（認証基盤の選定・導入）
- [ ] ユーザーごとの個人データ登録・保存（DB設計/権限）
- [ ] ユーザーごとの会話履歴の永続化（DB設計/権限）
- [ ] 個人データに基づいた漫画生成フローの実装

### 漫画仕様の推敲
- [ ] 漫画の仕様を推敲（コマ構成/文量/トーン/品質基準の再確認）

### 漫画生成の前段（要約文）推敲
- [ ] 漫画生成前の要約文作成ロジックを推敲（情報量/一貫性/口調）

### 漫画生成の安定性・永続化
- [ ] 画像の永続保存と配信（Cloud Storage）
	- API の data URL 返却をやめ、保存先 URL（署名付き）を返す


## P1（重要）

## 環境変数・Secrets
- [ ] 画像ストレージ用の環境変数を追加（バケット名/署名URL設定）
- [ ] AI/外部APIのSecrets管理（Secret Manager へ移行）

### フロントエンド拡張
- [ ] フロントエンドのデザイン改善
- [ ] 質問UIの追加と回答保存


### 監視・運用
- [ ] 漫画生成のジョブ監視（Cloud Logging + アラート）
- [ ] 失敗率・遅延の可視化（成功率/待機時間P95のダッシュボード）
